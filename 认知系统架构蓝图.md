**Project Yggdrasil (世界树项目) 架构蓝图**

## **一、 核心记忆引擎：确定性与物理隔离**

彻底抛弃模糊的向量检索，采用**逻辑寻址与图遍历**机制，将记忆结构打造为操作系统的文件管理层。

* **树图混合数据结构：**  
  * **树 (精度)：** 从宏观项目背景到微观代码细节的分层。不再存放超长文本，而是降维成“实体摘要 \+ 物理文件路径指针”。  
  * **图 (关联)：** 跨越时间和模块的实体逻辑连线。  
* **检索范式：猴子爬树 (Deterministic Traversal)**  
  * LLM 必须调用 explore(node\_id) 工具，从根节点逐层向下寻找目标，确保逻辑 100% 准确。  
* **节点版本控制 (Git for Memory)：**  
  * 抛弃就地覆盖修改。状态更新采用直接覆盖热数据，同时通过独立的 node-history 工具保留多版本快照，实现认知时间的漫游与回滚。  
* **量化软遗忘 (Decay Formula)：**  
  * 引入衰减系数 $k$。公式：  
    $$Weight \= Score\_{initial} \\times e^{-k \\cdot \\Delta t}$$  
  * 让 LLM 自己决定 $k$ 值。$k=0$ 为永久核心钢印，极大 $k$ 值为临时垃圾数据的“阅后即焚”。

## **二、 状态与上下文管理：无状态智能体**

解决超长上下文导致的“注意力涣散”与 Token 破产问题。

* **根节点即 PCB (进程控制块)：**  
  * 根节点强制记录三大元素：**我是谁**（认知角色）、**我在哪**（全局大纲地图）、**我要干什么**（当前执行状态与堆栈）。  
* **上下文频繁重启机制：**  
  * **硬重启：** 触碰 Token 预警线，强制挂起保存状态，清空历史重启，醒来阅读根节点接续工作。  
  * **动态记忆压缩 (思维链手术)：** 无法重启的深水区任务，主 Agent 声明“未来航向”，Sub-Agent 带着目标对臃肿的树节点进行外科手术级别的裁剪，保留无损的黄金思维链，替换掉主 Agent 脑中的旧上下文。

## **三、 执行通道：泛型工具与自反思回路**

打破 LLM 预装工具的数量限制，赋予其“阅读说明书使用任意工具”的能力。

* **动态工具分发 (Dynamic Tool Retrieval)：**  
  * 只有唯一的 MCP 接口 generic\_tool\_executor。LLM 在记忆树中查阅工具说明书后，通过此接口传入 target\_tool\_name 和 arguments\_json 执行。  
* **容错与自愈 (Self-Correction Loop)：**  
  * 剥离底层格式强校验。如果 JSON 报错，拦截错误日志并抛回给 LLM。  
  * 触发\*\*“主动怀疑”\*\*提示词：如果连续两次报错，LLM 必须停止重试，主动去查源码、查历史或发邮件求助。

## **四、 异步并发调度系统：数字员工车间**

放弃同步流式输出，建立事件驱动的并发工作流。

* **Reconnaissance (主动雷达扫描)：** 遇到巨大的长文件，主 Agent 不读，而是推入后台队列。  
* **Digester (认知同化 Sub-Agent)：** 在后台花费大量时间将长文本切块、摘要建树、提取实体连图。  
* **协程挂起与事件唤醒：** 主 Agent 遇到强依赖文件时挂起休息，Sub-Agent 消化完毕后通过事件总线将其唤醒。

## **五、 自我进化引擎 (Bootstrapping & RLAIF)**

用系统开发系统，防止演化停滞与模型坍塌。

* **沙箱与安全员：** Agent 运行于 Docker 沙箱。代码修改必须在临时分支进行，以**自动化测试 (TDD)** 作为能否落地的物理裁判。  
* **动态靶场 (Dynamic Benchmarking)：** 收集运行中的真实报错教训，自动生成新的基准测试题。  
* **A/B 评测法官 (Evaluator Agent)：** 主 Agent 修改工具源码后，必须跑通基准测试。法官 Agent 介入，依据“奥卡姆剃刀法则（代码简洁性）”和 Token 效率进行审批。  
* **求助中断：** 连续 N 次优化被驳回，触发 email\_human 向人类发送异步邮件，引入“高维范式转移”。
