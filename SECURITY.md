# Security Policy

## Supported Versions

Security updates are provided on a best-effort basis for the latest code in `main`.

## Reporting a Vulnerability

Please do not open public issues for security vulnerabilities.

Send a private report with:

- Summary of the issue
- Impact and threat model
- Reproduction steps or proof of concept
- Suggested remediation (if available)

Temporary contact method:

- Open a GitHub issue titled `SECURITY: request private contact` with no details.
- Maintainer will provide a private channel for full disclosure.

## Response Targets

- Initial acknowledgement: within 72 hours
- Triage update: within 7 days
- Fix timeline: depends on severity and complexity

## Scope Notes

Reports involving leaked local keys or misconfigured deployments may be treated as operational issues, but are still welcome for hardening guidance.
