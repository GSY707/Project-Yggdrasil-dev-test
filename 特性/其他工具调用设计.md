
### **1\. 报错重试（Self-Correction）**

当 LLM 输出了错误的 tool\_arguments\_json 时，控制流不应该直接崩溃，而是：

1. **捕获异常：** 你的底层代码尝试解析 JSON 或调用目标 API 失败，拿到详细的报错信息（例如 Error: Missing required argument 'location' in weather\_api）。  
2. **构建反馈：** 将这个报错信息打包成一条特殊的系统消息。  
3. **回传重试：** 将这条消息发回给 LLM，告诉它：“你刚才调用的工具失败了，这是错误日志，请阅读你查到的工具说明书，修正参数后重新调用。”
