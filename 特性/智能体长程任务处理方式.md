**工程化生命周期管理**：

### **1\. 启动期的“雷达扫描” (Reconnaissance & Scouting)**

Agent 应该主动探索。在接到一个长期大任务（比如重构一个模块）的初期，主 Agent 的第一个动作不应该是直接写代码，而是执行“环境勘探”。

* **目录树扫描 (Tree Scanning):** 主 Agent 调用系统层面的工具（如 list\_workspace\_files），获取整个项目的目录结构、文件大小和最后修改时间。  
* **依赖评估 (Dependency Triage):** 它扫描到有一个 50k\_api\_docs.md。由于它体积巨大且不在记忆树的索引中，主 Agent 会判定这是一个“高潜力的认知盲区”。  
* **预加载派发 (Proactive Dispatch):** 主 Agent 立即将这个文件推入后台任务队列，启动 Sub-Agent 开始长达几分钟的“建树+连图”同化工作。

### **2\. 执行期的“协程切换” (Context Switching)**

派发完同化任务后，主 Agent 并不需要傻等，它可以继续推进不需要这份文档的边缘工作（比如搭建测试环境、梳理现有的短小代码）。

### **3\. 强依赖下的“绝对阻塞” (Strict Blocking)**

派发完同化任务后，如果缺这内容不可，那只能等。

* **状态挂起 (Yield/Sleep):** 当主 Agent 推进到某一步，发现下一步必须调用那个正在被消化的 50k\_api\_docs.md 里的精准接口时，它会触发内部的状态机，将当前工作流标记为 BLOCKED\_WAITING\_FOR\_MEMORY。  
* **事件唤醒 (Event-Driven Wakeup):** 主 Agent 进入休眠，释放计算资源。直到 Sub-Agent 完成了那 5 万字的解析，更新了记忆树，并通过系统事件总线（Event Bus）发送了一个“记忆已就绪”的信号，主 Agent 才会被唤醒，沿着最高精度的记忆树继续极其精准地完成任务。
